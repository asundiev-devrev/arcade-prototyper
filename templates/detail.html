Webhook delivery fails silently for custom endpoints

When a webhook is configured to deliver to a customer-provided HTTPS endpoint, payloads that exceed the internal size threshold are silently dropped without any error surfaced to the user. The webhook delivery log shows a 200 OK status even though the payload was never transmitted to the external endpoint.

This affects all custom webhook integrations where the event payload exceeds approximately 256 KB, particularly common with audit log events and bulk operation results. Customers relying on webhooks for downstream automation are missing critical events.

ERROR 2026-03-10T14:22:08Z webhook/deliver.go:187 context deadline exceeded (Client.Timeout) endpoint: https://hooks.acme.io/devrev/events payload_size: 312847 bytes status: silently_dropped retry_count: 0

Expected behavior: delivery should fail with a visible error and trigger the configured retry policy (up to 5 attempts with exponential backoff). The webhook delivery log should reflect the actual failure status.
